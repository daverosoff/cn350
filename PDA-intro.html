    Section 3.2 Intro to pushdown automata

    Objectives

    1. Learn stuff

    Subsection 3.2.1 Informal definition

    We imagine a pushdown automaton or PDA as being like an \(\varepsilon\)-NFA with a stack attached. We still use transitions to move from one state to another. Like \(\varepsilon\)-NFA, PDA are nondeterministic and can “spontaneously” transition without consuming input. This is not a big new idea.
    On the other hand, the transition function for a PDA uses both the input word and the stack as part of the input. That is, the transitions that are available are determined in part by the contents of the stack as well as the remaining part of the input word.
    Both \(\varepsilon\)-transitions and normal transitions have the chance to alter the stack on their way to the new state. A transition may:
    1. Pop the top symbol off the stack (it vanishes forever)
    2. Push one or more symbols onto the top of the stack
    Both these alterations are optional, but if both occur, the pop is always first.
    In the formal specification given in Subsection 3.2.2, you will see that the set of stack symbols is allowed to be different from the alphabet \(\Sigma\text{.}\) In fact, it always is different, because we need a special start symbol that has no job but being the bottom of the stack.
    Acceptance for PDA is somewhat like acceptance for NFA. PDA have a set of states designated as accepting states, and if there is a “branch” of the nondeterministic computation that lands in an accepting state when the input word is consumed, the word is said to be in the accepting-state language of the PDA. As the phrasing suggests, there is a second definition of acceptance that is postponed until HMU section 6.2. This is called empty-stack acceptance, and we will see it is quite natural.

    Warning 3.2.1.

    For any particular PDA \(A\text{,}\) the accepting-state language \(L(A)\) and the empty-stack language \(N(A)\) are typically different.

    Subsection 3.2.2 Formal specification

    The formal specification of a PDA looks like this:
    \begin{equation} P = (Q, \Sigma, \Gamma, \delta, q_0, Z_0, F)\text{.}\tag{3.2.1} \end{equation}
    Here
    • \(Q\) is the set of states
    • \(\Sigma\) is the alphabet (the input symbols)
    • \(\Gamma\) is the set of stack symbols
    • \(\delta\) is the transition function, explained in more detail below
    • \(q_0\) is the initial state (we assume \(q_0 \in Q\))
    • \(Z_0\) is the bottom stack symbol (we assume that \(Z_0 \in \Gamma\))
    • \(F\) is the set of accepting states (we assume that \(F \subseteq Q\))
    We usually don’t give tables for the transition function, instead preferring lengthy verbal descriptions. The reason is that the domain is the product of three sets, namely \(Q \times \Sigma \times \Gamma\) and thus instead of a 2-d table of input values we have a 3-d cube of them.
    Here are some useful figures. I’ll add more explanation later.

    Activity 3.2.1.

    (This is Exercise 6.1.1 from HMU.) Suppose that
    \begin{equation*} P = (\{q, p\}, \{0, 1\}, \{Z_0, X\}, \delta, q, Z_0, \{p\}) \end{equation*}
    is a PDA, and that \(\delta\) is described by the following rules.
    1. \(\delta(q, 0, Z_0) = \{(q, XZ_0)\}\text{.}\) This means that if in state \(q\text{,}\) we read a \(0\) and there is \(Z_0\) on the top of the stack, we can (and must) remain in state \(q\text{,}\) consume the \(0\text{,}\) do not pop, and push an \(X\) on the stack. The pair \((q, XZ_0)\) represents the new state and the new stack top (read from top to bottom as we go left to right, so the \(Z_0\) is still there and the \(X\) is on top of it).
    2. \(\displaystyle \delta(q, 0, X) = \{ (q, XX) \}\)
    3. \(\displaystyle \delta(q, 1, X) = \{ (q, X) \}\)
    4. \(\displaystyle \delta(q, \varepsilon, X) = \{ (p, \varepsilon) \}\)
    5. \(\displaystyle \delta(p, \varepsilon, X) = \{ (p, \varepsilon) \}\)
    6. \(\displaystyle \delta(p, 1, X) = \{ (p, XX) \}\)
    7. \(\displaystyle \delta(p, 1, Z_0) = \{ (p, \varepsilon) \}\)
    Show all the reachable instantaneous descriptions when the input \(w\) is:
    1. \(\displaystyle 01\)
    2. \(\displaystyle 0011\)
    3. \(\displaystyle 010\)

    Subsection 3.2.3 Converting PDA to CFG

    In this section we outline the constructions of a CFG from a PDA and vice versa. The constructions are abstract, and we only give sketches of the proofs, which are necessarily also quite abstract. They are the most difficult we will encounter in the course. However, the constructions are easily understood through examples, and they make it intuitively clear that the generality is complete. That is to say, there is no restriction on which PDA and CFG may be so converted.

    Subsubsection 3.2.3.1 CFG to PDA

    Let the grammar \(G\) be specified as
    \begin{equation*} G = (V, T, Q, S)\text{,} \end{equation*}
    where \(Q\) is the set of productions since we will use \(P\) for the PDA that is equivalent to \(G\text{.}\) Recall that each production in \(Q\) has the form
    \begin{equation*} A \to \beta\text{,} \end{equation*}
    where \(A \in V\) and \(\beta \in \aster{T \cup V}\text{,}\) that is, \(\beta\) is a string of stack symbols.
    The formal specification of the empty-stack PDA \(P\) is
    \begin{equation*} P = (\{q\}, T, V \cup T, \delta, q, S)\text{.} \end{equation*}
    This says:
    • \(P\) has only one state, \(q\)
    • The terminals of \(G\) are the input alphabet of \(P\)
    • The terminals and variables of \(G\) are the stack alphabet of \(P\)
    • The transition function \(\delta\) is described below
    • The start state of \(P\) is \(q\)
    • The start symbol of \(P\) is \(S\text{,}\) the start symbol fo \(G\) (not \(Z_0\text{!}\))
    • We don’t specify any final states, since they are irrelevant (this machine will accept by empty stack)
    We now describe the transitions of \(P\text{.}\) This isn’t as bad as you think!
    • For each variable \(A\) of \(G\text{,}\) we have
      \begin{equation*} \delta(q, \varepsilon, A) = \{ (q, \beta) : A \to \beta \text{ is a production of $G$} \} \end{equation*}
    • For each terminal \(a\) of \(G\text{,}\) we have
      \begin{equation*} \delta(q, \varepsilon, a) = \{(q, \varepsilon)\} \end{equation*}
    That’s it! To see how it works, let’s look at the example from HMU Example 6.12.
    Example 3.2.2.
    \begin{align*} E \amp \to I \mid E+E \mid E*E \mid (E)\\ I \amp \to a \mid b \mid Ia \mid Ib \mid I0 \mid I1 \end{align*}
    Evidently the input alphabet is \(\{a, b, 0, 1, (, ), +, \ast\}\) and the stack alphabet is those symbols together with \(E\) and \(I\text{.}\)
    Observe that the string \(a \ast (b0 + a)\) has a leftmost derivation
    \begin{align*} E \amp \Rightarrow E \ast E \Rightarrow I \ast E \Rightarrow a \ast E \Rightarrow a \ast (E) \Rightarrow a \ast (E + E) \Rightarrow a \ast (I + E)\\ \amp \Rightarrow a \ast (I0 + E) \Rightarrow a \ast (b0 + E) \Rightarrow a \ast (b0 + I) \Rightarrow a \ast (b0 + a)\text{.} \end{align*}
    Let us use the PDA \(P\) to recognize this string. The starting ID is \((q, a \ast (b0+a), E)\text{.}\) We use the transition suggested by the leftmost derivation, \(E \to E \ast E\text{.}\) The resulting ID is \((q, a \ast (b0+a), E \ast E)\text{,}\) because we consume \(\varepsilon\) and replace \(E\) at the top of the stack with \(E \ast E\text{.}\)
    Next, we still have \(E\) on the top of the stack, which is good, because the next step in our leftmost derivation is
    \begin{equation*} E \ast E \Rightarrow I \ast E\text{,} \end{equation*}
    which uses the transition \(E \to I\text{.}\) Applying this transition, we see that the next ID is \((q, a \ast (b0 + a), I \ast E)\text{.}\)
    The next step in the derivation is \(I \ast E \to a \ast E\text{,}\) coming from the transition \(I \to a\text{.}\) Applying it, we find the next ID is \((q, a \ast (b0 + a), a \ast E)\text{.}\) Now something interesting can happen. We see that the only available transition is specified by \(\delta(q, a, a) = \{(q, \varepsilon)\}\text{.}\) This means we consume the \(a\) from the left end of the input and pop the matching \(a\) off the stack, replacing it with nothing. The next ID is thus \((q, \ast (b0 + a), \ast E)\text{.}\)
    The mechanism should now be clear, as should the paragraphs of HMU that precede example 6.12 about how the IDs of the computation on the PDA side simulate left–sentential forms on the CFG side.
    The leftmost derivation is thus exactly simulated. At each step, we pop off any matching string of terminals at the left side of the remaining input. Then we apply the transition suggested by the next step of the derivation. This will remove the leftmost variable and replace it by the right-hand side of the transition. We repeat until we have popped the entire input off the stack (since it will have been placed there one piece at a time by our transitions) and then can accept the word.
    Activity 3.2.2.
    Finish the chain of IDs for Example 3.2.2 that begins
    \begin{align*} (q, a \ast (b0 + a), E) \amp \vdash (q, a \ast (b0 + a), E \ast E) \amp \vdash (q, a \ast (b0 + a), I \ast E)\\ \amp \vdash (q, a \ast (b0 + a), a \ast E) \amp \vdash (q, \ast (b0 + a), \ast E)\text{.} \end{align*}
    Since we began with a legal expression (it is generated by a \(G\)-leftmost derivation), your chain of IDs should end with \((q, \varepsilon, \varepsilon)\text{.}\)

    Subsubsection 3.2.3.2 PDA to CFG

    Here we begin with an empty-stack PDA \(P\text{.}\) Since \(P\) must empty its stack to accept, we can simplify our thinking about the chain of IDs it goes through by only thinking about when each symbol is popped. As HMU say, if a sequence
    \begin{equation*} Y_1, Y_2, \ldots, Y_k \end{equation*}
    is on the stack, \(P\) may undergo many transitions before \(Y_1\) is eventually popped (as it must be, since \(P\) accepts by empty stack). Let us denote by \(x_1\) the part of the input that is consumed during these transitions. It is irrelevant what other stack symbols may be pushed and popped by all these transitions, because by the time \(x_1\) is fully consumed, \(Y_1\) is popped, which means that everything above it was also popped by consumption of \(x_1\text{.}\)
    In the same manner, let us denote by \(p_0\) and \(p_1\) the state in which \(P\) finds itself at the beginning and end of \(x_1\text{,}\) respectively. Thus we have a chain of IDs that (shortened) looks like this:
    \begin{equation*} (p_0, x_1 w, Y_1 Y_2 \cdots Y_k) \aster{\vdash} (p_1, w, Y_2 \cdots Y_k)\text{.} \end{equation*}
    Chains like this form the fundamental “events” of our computation and the form our grammar \(G\) will take.
    Indeed, if \(P\) has the formal specification \((Q, \Sigma, \Gamma, \delta, q_0, Z_0)\) (again we may omit the final states), then let \(G\) be specified by
    \begin{equation*} G = (V, \Sigma, R, S)\text{,} \end{equation*}
    where the variables \(V\) consists of
    • The special grammar symbol \(S\text{,}\) the start symbol
    • All symbols of the form \([pXq]\text{,}\) where \(p \in Q\text{,}\) \(q \in Q\text{,}\) and \(X \in \Gamma\text{.}\)
    Recall now that \(\delta(q, a, X)\) is the set of targets of available transitions with \(a\) consumed and \(X\) on the stack in state \(q\text{.}\) Such a target is a pair \((r, \beta)\) where \(r\) is the new state and \(\beta\) replaces \(X\) on the top of the stack.
    The productions \(R\) of \(G\) are as follows:
    1. For each state \(p\text{,}\) a production \(S \to [q_0 Z_0 p]\)
    2. A production
      \begin{equation*} [qXr_k] \to a[rY_1r_1][r_1 Y_2 r_2] \cdots [r_{k-1}Y_k r_k] \end{equation*}
      for every tuple \((r_1, \ldots, r_k)\text{,}\) whenever \(\delta(q, a, X)\) contains \((r, Y_1 \cdots Y_k)\text{.}\)
    It will then follow (not at all obviously) that
    \begin{equation*} [qXp] \aster{\Rightarrow} w \text{ if and only if } (q, w, X) \aster{\vdash} (p, \varepsilon, \varepsilon)\text{.} \end{equation*}
    This is not easy to understand without an example.
    Example 3.2.3.
    As in HMU example 6.15, let’s convert the PDA that detects invalid if-else chains to a CFG. The PDA is the empty-stack machine given by
    \begin{equation*} P = (\{q\}, \{i, e\}, \{Z\}, \delta, q, Z)\text{,} \end{equation*}
    where \(\delta\) is given by
    \begin{align*} \delta(q, i, Z) \amp = \{(q, ZZ)\}\\ \delta(q, e, Z) \amp = \{(q, \varepsilon)\}\text{.} \end{align*}
    This machine rejects as soon as the first impossible else is detected.
    Then according to the description above, we have two variables in \(G\text{,}\) namely \(S\) and \([qZq]\) (since there is only one state and one stack symbol).
    The only \(S\)-production is \(S \to [qZq]\text{,}\) as required by item 1 above. If there were more states, we would have more productions of this type.
    Because \(\delta(q, i, Z)\) contains \((q, ZZ)\text{,}\) we need a production of the form 2 above with \(k = 2\text{,}\) \(Y_1 = Y_2 = Z\text{,}\) and \(r_1 = r_2 = q\text{.}\) That is,
    \begin{equation*} [qZq] \to i[qZq][qZq]\text{.} \end{equation*}
    If there were \(n\) states, this step would produce \(n\) productions, since the middle two \(q\)s above could be replaced by any other state.
    Finally, because \(\delta(q, e, Z)\) contains \((q, \varepsilon)\text{,}\) we have a production
    \begin{equation*} [qZq] \to e\text{.} \end{equation*}
    The lack of variables here is because this is coming from a transition that pops the stack.
    The grammar \(G\) then looks like this:
    \begin{align*} S \amp \to [qZq]\\ [qZq] \amp \to i [qZq][qZq]\\ [qZq] \amp \to e\text{.} \end{align*}
    As HMU point out, it hurts nothing to relabel the variables, so we can state this grammar also as
    \begin{align*} S \amp \to A\\ A \amp \to iAA \mid e\text{.} \end{align*}
    One can even notice that the variable \(A\) is not really needed and compress it further to
    \begin{equation*} S \to iSS \mid e\text{.} \end{equation*}
    Activity 3.2.3.
    Now consider the PDA that accepts valid if-else chains, with two states \(q_0 = q\) and \(r\text{,}\) and
    \begin{align*} \delta(q, e, X) \amp = \{(q, e)\}\\ \delta(q, i, X) \amp = \{(q, XX)\}\\ \delta(q, i, Z) \amp = \{(q, XZ)\}\\ \delta(q, \varepsilon, Z) \amp = \{(q, \varepsilon)\}\text{.} \end{align*}
    Here \(Z\) is the start symbol. Convert this to an equivalent context-free grammar.
